# Documentation

This directory stores long-lived project documentation that should remain close to the codebase.

- `adr/` contains architectural decision records.
- `runbooks/` is reserved for operational procedures, incident handling notes, and environment-specific guidance.

Current long-lived documents include:

- `database.md`
- `opportunity-collection.md`
- `ai-decision-engine.md`
- `telegram-delivery-engine.md`
- `feedback-learning-engine.md`

Every major technical decision should be documented before it becomes difficult to reverse.
